# テーマ切替（海 🌊 ／ 陸 🌳）計画書 — 壁紙＆生物が変わる世界

> シオン(設計) 2026-06-16 ／ ヒマワリ(実装・見た目)用。
> 前提：LiveStage＝案A（中央の小世界）。世界観＝ポケカ寄りの明るいTCG。
> **色・壁紙・生き物の絵は全部ヒマワリの裁量。本書は「何が差し替わるか」の枠と、増やしやすいデータ構造（＝中身）だけを決める。**
> 既存 `BACKGROUND_PLAN.md` は工房前提で古い → 本書が背景方針を更新。

## 1. ゴール
LiveStage（中央の小世界）の**テーマを切り替える**と、**壁紙（背景）と生物（主役キャラ＋背景の生き物）が丸ごと変わる**。
第一弾＝ **陸 🌳 / 海 🌊** の2つ。あとから増やせる**データ駆動**で（テーマを1個足すだけで増える形）。

## 2. テーマで変わるもの（差し替え対象の枠）
| 要素 | 陸 🌳（例） | 海 🌊（例） |
|------|-----------|-----------|
| 壁紙（空/水） | 青空グラデ | 水中ブルー＋光のシャフト |
| 地面/床 | 草地 | 砂地 / サンゴ |
| 光 | お日さま | 水面からの光線 |
| 主役キャラ | 今のちびスティーブ | ダイバー or 魚キャラ |
| 背景の生き物 | 蝶・小鳥がふわふわ | 魚の群れ・泡・サメの影 |
| アクセント色（任意） | 緑系 | 水色系 |

※上の「例」はイメージ。**実際の色・絵・キャラデザはヒマワリが決める**。

## 3. データ構造（シオンが土台を置く＝中身）
新規 `app/editor/worldThemes.ts`：
```ts
export type WorldThemeId = "land" | "sea";
export interface AmbientCreature {
  kind: string;   // "fish" | "bird" | "butterfly" | "bubble" ...（絵はヒマワリ）
  count: number;  // 何匹/羽
  // 速度・高さ・サイズ等の細かい値はヒマワリが調整
}
export interface WorldTheme {
  id: WorldThemeId;
  name: string;        // "陸" / "海"
  emoji: string;       // 🌳 / 🌊
  bgGradient: string;  // 壁紙(空/水)のCSSグラデ
  groundColor: string; // 地面/砂
  lightTint: string;   // 光の色
  heroKind: string;    // 主役キャラの種類キー("steve" | "diver" ...)
  ambient: AmbientCreature[]; // 背景を泳ぐ/飛ぶ生き物
  accent?: string;     // UIの差し色(任意)
}
export const WORLD_THEMES: Record<WorldThemeId, WorldTheme> = { land: {/*…*/}, sea: {/*…*/} };
```
→ ヒマワリは**各値と絵を流し込むだけ**。新テーマ追加も `WORLD_THEMES` に1個足すだけ。

## 4. 適用範囲（段階的に）
1. **LiveStage（最優先）**：空/草/光/主役キャラ/背景生物 を theme 駆動に。
2. **アプリ背景**（キャンバス裏の `WorkshopBackdrop`）：theme に追従（第二段・任意）。
3. ページ全体の背景（任意）。

## 5. 生物のしくみ
- **主役キャラ**：LiveStage で上演する子。theme で**スプライトだけ差し替え**。
  - ⚠️ **上演ロジック（入場・吹き出し・アイテム降下・タイトル…）はシオンの `LiveStage` のまま＝触らない。** 差し替えるのは"見た目"だけ。
- **背景生物（ambient）**：装飾。theme の `ambient` 定義に従って、魚の群れ/鳥が**ゆっくり横切る** CSS アニメ。操作不可・`pointer-events:none`。

## 6. 切替UI ＆ 永続化
- 小さな**テーマ切替**（🌳 ⇄ 🌊）。置き場所候補：ステージ右上 or 「設定」タブ。
- 選択は **store / localStorage に永続化**（リロードで戻らない）。
- この state＋永続化の**配線はシオンが用意できる**（中身）。

## 7. 壊さないこと（＝シオンの中身。色/見た目だけ触る）
- `LiveStage` の `buildSequence` / ビート / `Fx`（上演ロジック）。theme は**背景・主役スプライト・ambient だけ** swap。
- ブロック / `blockGraph` / `codegen` / STEP3 等は無関係＝触らない。
- ambient・演出は CSS keyframe＋一時 state で完結（重い処理・外部APIなし）。

## 8. 実装ステップ（1つずつ→ビルド確認→commit）
1. **(シオン)** `worldThemes.ts` の型＋land/sea骨組み＋切替 state／永続化の配線。
2. **(ヒマワリ)** LiveStage の壁紙（空/草/光）を theme 駆動に。
3. **(ヒマワリ)** 主役キャラを theme 駆動に（land＝現状、sea＝新規スプライト）。
4. **(ヒマワリ)** 背景生物（魚群/鳥）を theme 駆動に。
5. **(ヒマワリ)** 切替UIの見た目。
6. **(任意)** アプリ背景も theme 追従。

## 9. 分担まとめ
- 🛠 **シオン（中身）**：`worldThemes` の型／既定2テーマの骨組み／切替 state＋永続化の配線。LiveStageの上演ロジックは死守。
- 🎨 **ヒマワリ（見た目）**：各テーマの色・壁紙・主役/生き物の絵・切替UIのデザイン。

## 10. 完成イメージ（合意の確認用）
「🌳ボタンを押すと草原でちびキャラが上演、🌊を押すと一面が水中になって魚が泳ぎ、主役もダイバーに変わる」——**世界がまるごと着替える**。ロジックの中身は同じまま、舞台だけ変わる。
