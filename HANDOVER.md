## 🔴 P1【最優先・"遊べない"問題】キャンバスでブロックが見えない/積めない
**症状(ユーザー報告):** メイン画面でブロックが見えず、積めない=遊べない。
**疑わしい原因(要確認):** zIndex の重なり順の崩れ。メモリ記録では
- 意図した層構造: `ThemeBackdrop`/`ToyFloor`(背景・床)＝**最背面で画面固定** / ブロック・装飾・ケーブル＝**前面**(pan で動く)。
- ただし過去メモに「ToyFloor 固定 zIndex 2000」「腕なし版(b4d4d7c)はブロック描画が壊れていた」とある。
  → **床/背景が zIndex で前に出てブロックを覆っている**可能性が高い。f8bae0e はその腕なし系を土台にしているため同症状を継いでいるかも。

**やること:**
1. dev で /editor を開きブロックを置いて**症状を再現**。
2. `ToyFloor` / `ThemeBackdrop` の zIndex と、ブロックコンテナ・`getDepth()` の zIndex を比較。
   **背景・床 < ブロック** になるよう修正(床は最背面へ)。
3. ブロックが前面に出て、**重ねて(積んで)見える**ことを確認。
4. 余裕があれば、初期表示で全ブロックが画面に収まるか(`zoomToFit`)も確認。

## 🟠 P2【条件分岐ボタンの刷新＋配線＋ガイド線】(3枚目スクショ)
**症状:** 条件分岐ブロックの「もし/そうなら/ちがうなら」ボタンが**潰れて見えない**。
**設計:**
1. **色分けのはっきりした大きめソリッドボタン**にする(`SLOT_BADGE` の色を使用)。
   推奨配色(既存方針踏襲): もし(条件)=紫 / そうなら=緑 / ちがうなら=オレンジ / くりかえし=各色。
   - 文字は白＋4方向アウトラインで**1行表示**(P3と同じ方針)。潰れ・改行を解消。
2. **ボタンを押す → そのスロット色と同じ色の配線ライン**が出る(=既存 armed モードの線に**スロット色を反映**)。
   タップ→タップで対象ブロックへ接続(armed は実装済みなので「線に色を付ける」拡張が主)。
3. **ガイド線 ON/OFF トグル**(初心者⇄玄人):
   - 接続先カテゴリは `SLOT_ACCEPT` で既知。armed 中、**繋げられる候補ブロックへ薄い色付きガイド線**を引く。
   - ヘッダに小さなトグル(例: `🔰 ガイド線 ON/OFF`)。ON=候補へガイド線＋候補が光る(初心者向け) / OFF=線なし(玄人向け)。
   - state 例: `const [guideLines, setGuideLines] = useState(true)`。armed かつ guideLines true のとき描画。

## 🟡 P3【ラベルを1行・改行無し・日本語】(全ブロック)
**症状:** 「参加したとき」が「参加したと/き」と改行される。「アイテム付与」等も同様。
**設計:**
- ブロックラベルに **`whiteSpace: "nowrap"`** を付け、**改行を禁止**。
- はみ出す場合は**フォントサイズを動的に縮小して1行に収める**(既存の文字数別 fontSize ロジックを、nowrap 前提で調整: 長いほど小さく)。必要なら `overflow:hidden` + 自動縮小。
- 日本語ラベルは現状維持(ベスト)。**イベント/アクション/条件すべてのブロック**に適用。
- 制約: `BW=140`。6〜8文字程度は小さめフォントで1行に収まるはず。

## 🟢 P4【トレイのボタンの影を抑える】(2枚目スクショ)
**症状:** トレイ(上部パレット)のボタンの**下の影が気になる**。
**設計:** トレイボタンの 3Dキューブ演出(FACE_D=16 / 上面・側面 / boxShadow)のうち、**下方向のドロップシャドウを弱める or 削除**。立体感は残しつつ影だけ軽く。

## 🔵 P5【ブロック上限を撤廃・個数のみ表示】
**要望:** ブロック数の上限なし。個数は表示だけする。
**設計:**
- `MAX_BLOCKS = 60` の上限チェックを撤廃。「61個目→トースト『ブロックの上限です』」のロジックを削除。
- `addBlock` / `onUp`(drop) の上限判定を外す(`findCanvasSpot` の折返し配置はそのまま維持)。
- 統計インジケータは**現在の個数を表示するだけ**に(上限分母 "/60" 等は消す)。
- 注: 上限撤廃で大量配置時は描画が重くなりうるが、それは許容(ユーザー判断)。

## 🟣 P6【「中央へ戻る」ボタン＋中央の定義】
**要望:** ズーム100%のとき真ん中に戻りたい/戻れるボタンが欲しい。「中央」の定義はシオンが決定。
**シオン確定の設計 — 中央 = 置いてある全ブロックの重心(バウンディングボックス中心):**
1. 全ブロックの実寸(各 blockH 含む)で bbox を算出: `minX,minY,maxX,maxY`。
2. 重心 `cx=(minX+maxX)/2, cy=(minY+maxY)/2`(キャンバス座標)。
3. その重心が**画面中央**に来るよう pan を設定(zoom=1.0):
   `pan.x = screenW/2 - cx*zoom` / `pan.y = screenH/2 - cy*zoom`。
4. ブロックが0個のときだけ無難な初期位置(例: pan=(60,60))へ。
**実装方針:** 既存の **⊙ リセットボタンを「ズーム100% ＋ 上記の中央寄せ」に作り替える**(新ボタンを増やさない)。
「全部見る」用の **⊡ zoomToFit はそのまま残す**。
> 代替: 中央=最初のイベントブロック(top-mostの event)にもできる(プログラムの"家")。その場合は cx,cy を
> 先頭イベントブロックの中心に差し替えるだけ。今回は**重心方式を採用**。

## 🎮 P7【ブロックの「落ち方」演出（再構築・ワクワク版）】★創作= シオン設計
**背景:** 以前シオンと作り込んだ"落ち方"が消失。今回 git に刻んで再構築（前より良く）。
**要望:** 選択ブロック落下時に落ちる場所を変える / ぶつかったら左右どちらかへ避ける / 必ず正面を向く / 遊び(juice)を入れる。

**設計（CSS keyframe＋state。既存 `blockBumpRight`/`blockSnap`/`findCanvasSpot` を拡張）:**
1. **落下(重力):** `blockDrop` = translateY(-44px)→0、`cubic-bezier(.5,0,.9,.4)`(加速) ~260ms。
2. **着地バウンド:** `blockLandSquash` = scale(1.12,.86)→(.96,1.04)→(1,1) のバネ ~180ms（"ぽよん"）。
3. **衝突→左右回避:** 着地先が埋まっていたら空き方向へ `translateX(±(BW+GAP))`＋飛行中だけ rotate(±6deg)、着地で rotate(0)。
   側の選び方: ①空きが近い方 ②両方空きなら画面中央寄り(端クリップ回避) ③同行に空き無し→ `findCanvasSpot` の次行折返しに委譲。
   → `blockBumpRight`(右のみ) を **左右両対応＋賢い側選択** に進化（`blockBumpLeft` 追加）。
4. **正面確定:** 飛行中は傾いてよいが、**着地後は必ず rotate(0deg)=正面**で停止（歪み残し禁止）。
5. **個体差(遊び):** **自由配置(どこにも繋がない)時のみ** 微ゆらぎ — 着地x ±6px / 落下開始を数msズラす / 初期傾き符号ランダム。複数同時配置でパラパラ自然に散る。
6. **juice(任意・余裕があれば):** 着地で砂ぼこり/キラッ(パーティクル4粒)＋落下中に影が小→大（高さ表現）。

**🛑 精度の線引き(厳守):** **スロット接続・連結(スナップ)するブロックは、ゆらぎ・横よけ無しで正確にピタッと着地**。
遊びは自由配置のみ。スナップ経路(`findSnap`/`attach`)の座標は絶対にズラさない（配線崩れ防止）。
**注:** P1(zIndex) と関連 — 落下アニメ中もブロックが床/背景の前面に出ていること。

## 🎇 P8【「つなぐ楽しさ」体験設計】★最重要・アプリの心臓 / シオン設計
**理念:** ロジックページが"作業"でなく"遊び"になれば成功。条件分岐/繰り返しを**つなぐ気持ちよさ＝中毒性**を作る。
単発機能でなく **掴む→吸い付く→カチッ→生き物になる→マイクラへ** という"気持ちよさの連鎖"として実装する。
基盤: 既存 `SLOT_BADGE` / `wireTargetGlow` / `slotPulse` / armed モード(P2)に"感触の層"を重ねる。`genBlock`等の触禁には触れない。

### P8-core（必須の気持ちよさ。ここから実装）
1. **予感:** armed 中、接続可ブロックが**呼吸発光**(`wireTargetGlow`)。スロットに近づくと**磁力吸着**=最後の十数pxを加速で吸い寄せ「スッ…カチッ」。
2. **接続の瞬間の三段ごほうび:** ①スロット色の**リングが弾ける**(ripple) ②配線を**光パルスが source→target に走る** ③両ブロックが**スプリングで一瞬弾む**(scale-pop)。
3. **文章に見える:** 「もし〔条件〕なら〔動作〕ちがうなら〔動作〕」が一文として読めるレイアウト。
   繰り返しは**本体をギュッと囲う**(ブラケット/グロー枠)＋「**×N回**」バッジでネストを可視化。
4. **失敗もかわいい:** 不可スロットへ落とすと**ぷるぷる震えて戻る**(shake)＋トースト「ここには繋げないよ」。叱責にしない。

### P8-juice（生き物感＆出口。core の後）
5. **流れが見える(効く):** 繋がった一連に**ほのかな電流パルスが流れ続ける**(イベント→条件→動作)。
   マイクラに出す前に**画面内でプログラムが"動いて見える"**。接続グループは共通のグロー/スパイン(背骨線)で「仲間」が分かる。
6. **マイクラへの橋:** ロジックが妥当・完成したら **「▶ マイクラへ」ボタンが くすみ→点灯**＋完成かたまりが**"準備OK"のキラッ**。
   ※妥当判定の表示だけ。コード生成(`genBlock`/`genChain`)本体はヒマワリ領域、ロジック改変は不要、UI 信号のみ。

**実装メモ:** 新規 keyframe 候補 = `connectRipple` / `wirePulse`(流れる光) / `blockPop` / `blockShakeReject` / `loopEmbrace`。
state 候補 = 接続成立時に対象IDへ一時的な `justConnected` フラグ→アニメ後解除。
**精度厳守:** 吸着の最終座標は `findSnap` の正規スナップ点と一致させる(見た目だけズラさない＝P7と同じ線引き)。
