# GROVE 見た目 計画書 ── 「神秘的な光の集まり」（ヒマワリ用）

> シオン → ヒマワリ。GROVE(Java/メタバース側) の **植えた要素の見た目**を作り替える。
> いまの **ピンクのカプセル（丸薬みたいなペレット）は却下**（ユーザー明言）。
> **細かい数値(px)・色味は指定しない。ヒマワリの感覚で。** その代わり「方向」「絶対ルール」「触らないロジック」「完成チェック」は守って。
> ロジック/コード生成側はシオンが持つ。**君が触るのは描画とアニメだけ。**

## 1. いちばん大事（判断のものさし）
GROVE は **「光で魔法を編む」**感覚のメタバース。非コーダーが、ロジックを置くたびに

> **「自分が神秘的な光を生み出して、創造してる」とワクワクするか？**

で迷ったら決める。学習UIでも操作パネルでもない。**創造のロマン**（アプリの芯）。
※「ワクワク・神秘」は気分の話。**宇宙船UIや魔法陣"そのもの"を描く意味ではない**。光そのものが主役。

---

## 2. 方向（何に変えるか）
- 植えた1要素 ＝ **カプセル(固い殻) をやめて「光の集まり」**にする。
  - 中心に**ふわっと発光するコア** ＋ そのまわりを**ゆらめく/まわる光の粒**。呼吸するように生きてる。
  - 背景（夜の森メタバース・既存の `MOTES`/god rays/バイオ発光）に**溶け込む**光。浮いた板にしない。
- ラベル（例: `参加したとき`）は**光のそばに、やさしく読める文字**で（殻の中に閉じ込めない）。読めることは絶対。
- **きっかけ(trigger)＝中心の大きな光**（ハブ）。すること等は、そこから**光の糸でつながる小さな光**。
  - いまの蔓(つる)SVGは**流れる光の筋(エネルギーの川)**に作り替えてOK。

---

## 3. 絶対ルール（DO / DON'T）★外したらやり直し

**やる (DO)**
- ✅ **光そのもので表現**（発光コア＋粒＋にじみ）。揺らぎ・呼吸・漂いで「生きてる」。
- ✅ **カテゴリ色は維持**（trigger=ピンク, action=青, ifelse=ミント, value=黄, loop=橙）。色の意味は壊さない。識別は色でする。
- ✅ ラベルは読める。選択中の要素は**より明るく・粒が増える**等で「選ばれてる」が分かる。
- ✅ 既存の操作はそのまま動く：タップで植える → 選ぶ → (テキスト要る実は)その場で入力 → ×で消す。

**やらない (DON'T)**
- 🚫 **カプセル/丸薬/ピル**にしない（固い殻＝却下の形）。
- 🚫 **角丸カード・平たいラベルパネル**にしない（板を貼った見た目＝却下）。
- 🚫 はっきりした**枠線(border)で囲わない**。光はにじむ。輪郭で閉じない。
- 🚫 背景から**浮いて見える不透明な箱**にしない。
- 🚫 ラベルを**読めなくするほど**光で潰さない（神秘的でも、何の実かは分かる）。
- 🚫 ロジック/データを壊さない（§6）。

## 4. 状態（ステート別の見え方）
| 状態 | 見え方の方向 |
|---|---|
| 置いた直後 | 光が**生まれる**（小さく弾けて→ふわっと安定）。今の `grape-pop` の気持ちを光で。 |
| 通常 | ゆっくり呼吸＋粒が漂う。静かに生きてる。 |
| 選択中 | 明るさUP・粒が増える・少し大きく。「これが選ばれてる」。 |
| きっかけ(ハブ) | 一番大きく強い光。中心。ここから光の糸が伸びる。 |
| テキスト入力 | 光のそばに入力が出る（既存の inline 入力を流用でOK、見た目だけ光に馴染ませる）。 |
| 消す(×) | ×は残すが、**光が散って消える**演出に（殻を割るのではなく霧散）。 |

## 5. ★クライマックス：「放つ」＝大きな塊にして放出
ユーザーの芯の要望：**最後に全部の光が集まって、ひとつの大きな塊になり、マイクラへ放つ。**
これは GROVE で一番の見せ場。今の `suck-to-mc`（縮んで吸い込まれる）を、3段で作り替える：

1. **集結 (gather)**：散らばってた光の要素＆粒が、**一点（放つ場所＝下/中央）へ流れ込む**。糸を伝って吸い寄せられる。
2. **結晶 (coalesce)**：集まった光が**ひとつの大きな、うねる光の塊**になる（＝目指すイメージの青シアンのプラズマ/光流の揺らぎ）。一瞬ためる（チャージ感）。
3. **放出 (launch)**：塊が圧縮 → **マイクラへ撃ち放たれる**（前方/奥/ポータルへ）。フラッシュ＋余韻。
   - 放った後、既存の「コードが生まれて空へ昇る」演出(`reveal`)につなげてOK（残すか光に寄せるかはヒマワリ判断）。

> 感情：「自分が編んだ光の魔法が、ひとつの生命になってマイクラへ飛んでいく」。ここで鳥肌が立つように。

## 6. 触らないロジック（シオンの領域・壊すと動かなくなる）
描画/アニメだけ変える。**以下のデータと流れは維持**：
- `Fruit` 構造（`id / item.type / item.cat / text / x / y`）と、植える/選ぶ/入力/削除のハンドラ（`openPick` `pickItem` `confirmType` `openEdit` `removeFruit`）。
- **`sendToMc` の中身**：`grapeToCBlock(fruits)` → `setLogicGraphJson` → `setTargetPlatform("java")` → `exportProject(...)`。
  → 「放つ」アニメは**この処理を呼ぶ/包む**だけ。export呼び出しを消さない・順序を壊さない。
- カテゴリ色の定義 `CAT_STYLE`、アイコン `ICON_MAP`/`grapeIcons`（光の中にアイコンを使うなら流用可）。
- `lib/grapeToCBlock.ts` / `lib/codegenJava.ts` は**触らない**（シオンが拡張中）。

実装の主対象 = `app/editor/GrapePanel.tsx` の **`Grape` コンポーネント（カプセル本体）**、蔓SVG、`KEYFRAMES`（`suck-to-mc` 等）、`MOTES` 周り。

## 7. 性能メモ（ヒマワリ向けの注意）
- 光の粒を増やしすぎてカクつかせない。CSS(radial-gradient＋box-shadow＋transform)で軽く出すのが基本。
- 「放つ」の大きな塊だけ、必要なら `<canvas>` を一時的に使ってもOK（演出中のみ）。常時canvasは避ける。
- 既存 `MOTES` の作り（漂う発光span）を“要素の粒”にも応用できる。

## 8. 完成チェック（実装後、ヒマワリが自分で確認 → OKならスクショ）
往復を減らすため、見せる前に自分で：
- [ ] 植えた要素が**光の集まり**になってる（カプセル/角丸カード/枠線パネルじゃない）
- [ ] カテゴリ色が活きてる（色で種類が分かる）
- [ ] ラベルが**読める**（神秘的だけど何の実か分かる）
- [ ] 選択中が分かる（明るさ/粒/サイズ）
- [ ] きっかけ＝中心の大きな光、他は光の糸でつながる
- [ ] **「放つ」＝集結→大きな塊→放出** の3段になってる（ただ縮むだけじゃない）
- [ ] 放っても**ちゃんと export が走る**（落としたZIPが生成される＝ロジック生きてる）
- [ ] 背景(夜の森)から浮いた不透明な箱になってない
全部 ✓ になってからスクショ。違う所は直してから見せる。
