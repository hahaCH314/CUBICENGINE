# GROVE 見た目 計画書 ── 「生命の樹（光のツリー）」（ヒマワリ用）

> シオン → ヒマワリ。GROVE(Java/メタバース側) の **ビジュアルと操作設計を全面刷新** する。
> 以前の「マインドマップ型 / 天体型（ハブ＆放射状）」はプログラミングのシーケンシャルな制御フローと合致せず、手動配置が使いにくかったため廃止し、**縦に成長する「生命の樹（ツリー）モデル」** へ一新する。

---

## 1. 核心コンセプト（判断のものさし）

GROVE は **「生命を育み、光の魔法を編む」** 感覚のメタバース。
プログラムを組むたびに、

> **「自分が神秘的な光の木を育て、命を生み出している」とワクワクするか？**

で迷ったら決定する。
簡単モード（SPROUT）が横に並べる **「カードソリティア」** なら、プロモード（GROVE）は縦に伸びていく **「生命の樹（ツリー）」** として、アプリ全体の対比としても美しく整理する。

---

## 2. ビジュアル ＆ 操作仕様（生命の樹モデル）

### 🌳 構造（木と幹）
- **根っこ / 双葉（ Trigger ）**: 
  ハブ（トリガー）をプレイエリアの最下部中央に「根っこ」として配置します。ここがすべての生命の起点となります。
- **光の幹（ Action / Loop / ... ）**:
  アクションを追加するたびに、上に向かって幹が伸びるように、光のオーブ（実）が自動的に縦一列へ積み重なっていきます。
- **枝分かれ（ If / 条件分岐 ）**:
  `co_if`（もし〜なら）が挿入されると、幹が左右に美しく「二股に枝分かれ」し、それぞれの枝（then枝 / else枝）がさらに上に向かって伸びてしていきます。

### ✋ 操作（自動整列 ＆ ドラッグ並び替え）
- **自動レイアウト（スナップ不要）**:
  ラジアルメニューからオーブが生まれると、ドラッグで位置を合わせる面倒な操作はなく、自動的にツリーの次の「節」としてカチッと最上部（または指定の挿入位置）に自動整列します。
- **ドラッグで順序スワップ**:
  すでにツリーにあるオーブを上下にドラッグすることで、順番を滑らかに入れ替えることができます。
- **引き抜いて削除**:
  オーブをツリー（幹）から大きく横に引き抜くか、✕ボタンを押すことで、光が霧散して消え、前後が自動的に再連結されます。

### ✨ 光と繊維の表現
- **にじむ精霊 of 光（オーブ）**:
  カプセルや不透明な枠線を廃止し、柔らかくにじむバイオ発光オーブ（ウィスプ）にします。周囲を漂う極小の光の塵（胞子）が呼吸するように明滅します。
- **幹をつなぐ「光の繊維」**:
  触手っぽさを完全に消した、極細で流麗な「光の繊維」が幹として各オーブを貫き、下から上へ向かって実行エネルギーパルスが絶えず流れます。
- **年輪ガイド**:
  ハブを中心に、木の年輪を模した極薄の同心円ガイド（呼吸するように明滅する波紋）を背景に配置します。

## 3. クライマックス：「放つ」＝ツリーのエネルギー収束と放出

ツリーが完成し、「放つ」ボタンを押した際の見せ場アニメーション：

1.  **集結 (gather)**: 
    ツリーのすべての枝やオーブから光の粒子が、根っこ（Trigger）に向かって光の繊維を伝って一気に流れ落ちていきます（エネルギーの根への収束）。
2.  **結晶 (coalesce)**: 
    根っこの位置で、集まったエネルギーが激しくうねる巨大な「プラズマの種（結晶）」に融合し、まばゆく明滅チャージします。このとき効果音が鳴り響きます。
3.  **放出 (launch)**: 
    チャージがMAXに達すると、結晶が瞬時に収縮したのち、画面全体のホワイトフラッシュと共に天（奥）に向かって撃ち放たれ、その瞬間にZIP（Java MOD）がダウンロードされます。
4.  放出の余韻のなかで、静かにコードのストリーム（`reveal`）が空へと昇っていきます。

## 4. 触らないロジック（壊すと動かなくなる）

- `lib/grapeToCBlock.ts` 和 `lib/codegenJava.ts` は変更せず、`fruits` 配列のデータ順がそのまま `nextId` 鎖に変換される構造を利用します。
- そのため、ツリーの自動レイアウト（`x`, `y` 座標の自動計算）は、`fruits` 内の Trigger から始まる親子関係（CBlock的な鎖）に基づいて、コンポーネントレンダリング時（または `fruits` 更新時）に自動算出する方式をとります。
- ＝ `setFruits` では親子順（配列の順番やブランチ構造）だけを管理し、実際の描画位置 `x`, `y` は数式でクリーンに自動決定します。
