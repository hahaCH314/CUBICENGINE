## ⚠️ 負け筋（＝ガードレール。ここを踏んだら死ぬ）
1. **「作る→マイクラに入る」が折れる** → 何をしても無。最優先で守る。
2. **最初の5分で"わかりにくい"** → 楽しさの堀が発動しない（過去の指摘の再来）。
3. **機能を薄く広げる** → MAMに物量で負ける土俵に自ら乗る。深さ＜広さは禁物。
4. **物語・値観を隠す** → 唯一無二の武器（12歳の本物の動機・無料・無広告）を捨てる。

## ロードマップ（上から順に効く）

### Phase 0 — table stakes（今すぐ潰す。これ無しに先は無い）
- [ ] **E2E検証：スマホで「作る→シェアシート→Minecraftインポート」が本当に通るか実機確認**（担当：伊波さん実機／シオン確認）。※コード/ビルドはOK、実機は未検証。
- [ ] **本アイコン差し替え**（発注済：アクアのキューブ核・MAM差別化）。担当：ヒマワリ→シオンレビュー。
- 勝ち指標：初見の人がスマホだけで1個アドオンを完成→導入できる。

### Phase 1 — 堀を立てる（最初の5分の"面白い"）
- [ ] **オンボーディング**：リンク→最短で「動く1個」が手に入る導線。迷子を作らない。
- [ ] **"wow"作品を3〜5個に絞って完璧に**（子どもが本当に欲しい物＝例：ペット/武器/一撃モブ等）。広げず、各1個の完成体験を最高に。
- [ ] **お祝い演出＝魔法の瞬間**を磨く（完成→コード誕生→マイクラへ、の高揚）。
- 勝ち指標：初回セッションで「もう1個作りたい」と思わせる（再訪/継続）。

### Phase 2 — 非対称の入口を活かす
- [ ] **摩擦ゼロ funnel**：A2HS（ホーム追加）UX、iOS手動追加の軽い案内。
- [ ] **作品をスマホで遊ぶ/見せる導線**（作った物の共有・自慢）。
- 勝ち指標：ストア無し・アカウント無しで「入れた/遊べた」まで到達する率。

### Phase 3 — 物語で"発見される"（PWAはストア検索順位が無い前提）
- [ ] **物語主導の配布**：本物の動機・作り手の顔（安全配慮済）・無料無広告を前面に。
- [ ] **YouTube / creators / Discord** 導線（既存Discord/Ko-fi活用）。
- [ ] ブランド一貫性：アクアのキューブ核・光の粒・ワードマークを全面で統一。
- 勝ち指標：紹介1本から流入→完成まで回る。

### Phase 4 — 北極星（MAMが一生来ない場所）
- [ ] **GROVE/Java＝最強modづくりIDE**の深化（前提mod/GeckoLib/Blockbench/IDEA取込 等）。
- 勝ち指標：「本格modを作りたい人」がIntelliJより楽しいと感じる。
