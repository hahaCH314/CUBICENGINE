# 公開までのロードマップ（伊波さん用・やさしい言葉）

> このアプリを「ローカル/オフラインのアドオン作成ツール」として、安全に公開するまでの道のり。
> 専門知識は要りません。**各ステップに「あなたがやること」「シオン(AI)がやること」「安全の確認」** を書いてあります。

---

## 🟢 いまの現在地（もう出来ていること）
- ✅ アプリはローカルで動く（`http://localhost:3000`）。
- ✅ **ログイン不要**で使える（アカウント機能は外した＝危険な部分が無い）。
- ✅ 保存は**あなたのPC内**（localStorage）。サーバーに何も送らない。
- ✅ アドオン書き出し（.mcaddon）は**アプリの中だけ**で完結。
- ✅ **デスクトップアプリ化の土台あり**（Electron）。マイクラ自動検出・ビルドまで実装済み。
- ➡️ **つまり「中身」はもうある。** あとは仕上げ・テスト・配布。

---

## 🗺️ これからの5ステップ

### STEP 1. 中身を仕上げる（機能・見た目）
- **あなた**：画面を見て「ここ違う/いい」を言うだけ。**芸術はあなたにしか見えない**ので、ここが主役。
- **シオン**：言われた所を実装＆コミット。
- **安全の確認**：見た目はあなたの目でOK判定。コードは毎回ビルドが通ってからコミット。

### STEP 2. 「本物のアプリ」として自分のPCで動かす
- **あなた**：シオンが用意した手順で起動ボタンを押すだけ。
- **シオン**：デスクトップアプリ（Electron）を起動する手順を用意。`npm run electron:dev`。
- **安全の確認**：ウィンドウが開いて、ブラウザ版と同じに動けばOK。

### STEP 3. 実際にアドオンを作ってマイクラで試す（←一番大事な安全網）
- **あなた**：アプリでブロック/ロジックを作る → ダウンロード → **自分のマイクラに入れて動かす**。
- **シオン**：出力の不具合があれば直す。
- **安全の確認**：⭐**「マイクラで本当に動いたか」を、あなた自身が必ず確認する。** これが、AIの嘘（ハルシネーション）を防ぐ唯一にして最強の方法。動けば本物、動かなければ嘘。

### STEP 4. 配布用にパッケージ化（インストーラを作る）
- **あなた**：シオンに「パッケージ作って」と言う → 出来たファイルを受け取る。
- **シオン**：`npm run electron:build` で Windows 用インストーラ(.exe)を作る手順を実行/案内。
- **安全の確認**：出来た .exe を**自分のPCにインストールして動くか**を確認してから配る。

### STEP 5. 公開（配る）
- **あなた**：出来た .exe を配布サイトに置く。おすすめ：
  - **itch.io**（ゲーム/ツール配布に最適・無料・簡単）
  - または GitHub の「Releases」
- **シオン**：説明文・スクショの文面を手伝う。
- **安全の確認**：ローカルアプリなので、**個人情報もサーバーも無い＝事故の危険が最小**。安心して配れる。

---

## 🛡️ ずっと守るルール（あなたの武器）
1. **「できた」を信じない。現実で確かめる。** アドオンは必ずマイクラで試す。
2. **芸術はあなたが判定。** AIには見えない。あなたの目が最終決定。
3. **危険な物は足さない。** アカウント/サーバー/課金/個人情報＝公開しない方針（決定事項）。足したくなったら、その時は人間の専門家に見てもらう。
