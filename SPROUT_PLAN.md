# SPROUT 一新 計画書（ヒマワリ向け）

作成: シオン / 2026-06-14 / 依頼: 伊波さん
対象: 統合版 **SPROUT**（= `app/editor/LogicPanel.tsx` の見た目／プレイ面）＋ `ModelPanel` / `SettingsPanel` のモード選択。

---

## 0. 一言
SPROUT を **「カジノの華やかさ × でもギャンブルじゃない・明るく健全なゲーム感」** に一新する。プレイ感は **ソリティア風**。芯は [作る歓び]（`core_soul_joy_of_creating`）＝学習でも機能網羅でもなく「作って楽しい」。

## 1. トーン（ここ超重要）
- **採用**：カジノ的な華やかさ＝**光・きらめき・コイン的なツヤ・小気味よい効果音・気持ちいい回転/めくり**。明るくポップ。
- **NG（ギャンブルに寄せない）**：賭け・お金・チップ・確率/射幸を煽る演出・「当たり/ハズレ」。**運要素で煽らない**。あくまで「自分で選んで作る」が主役。
- 子供にも安心、でも玄人もワクワク（[feedback-craft-not-dumbed-down]）。古いUI定番に逃げない（[feedback-novelty-over-familiar]）。

## 2. レイアウト（3カラム）
```
┌── 左：スロット ──┬──── 中央：プレイ面（ソリティア風） ────┬── 右：操作 ──┐
│ イベント        │                                        │ クリア      │
│ アクション      │   選んだアイテムを並べて組む（カード的） │ 保存/読込   │
│ 条件 / 値 …     │                                        │ コード      │
│ (リールで回す)   │                                        │ ヘルプ 等   │
└────────────────┴────────────────────────────────────────┴────────────┘
```
- **左＝スロット（リール）**：イベント等のカテゴリ/アイテムを **スロットマシンのリールのように回して選択**。回すと候補がカチカチ切り替わる気持ちよさ。選んだら中央へ置ける。
- **中央＝プレイ面**：**ソリティア風**にカード（＝アイテム）を並べて組み立てる。トランプを並べる手触り（めくり/スライド/ピタッと収まる）。
- **右＝操作ボタン**：クリア・保存/読込・コード・ヘルプ等を **右側にまとめて、分かりやすく可愛いボタン**に。装飾的（飾り兼用）でOK。lucide アイコン（既に導入済み）活用可。

## 3. スロット（左）の挙動イメージ
- カテゴリ＝リール。**上下に回して**目的のアイテムに合わせる（スロット/ホイール）。
- 「カチッ」と止まる手応え＋軽い効果音。**運ではなく"選ぶ"**（自分で回して決める）。
- 決定でそのアイテムが中央のプレイ面へ。

## 4. モード選択の整理（ModelPanel / SettingsPanel）
- モデル画面が今 **2面選択**になっている → ラベルを **「簡単 = SPROUT」「プロ = GROVE」** に変更。
- **設定画面も同様**に「簡単=SPROUT / プロ=GROVE」へ。
- （※`起動`タブの削除は page.tsx 側＝シオン担当。下記参照。）

## 5. 🚫 触禁（心臓＝ロジック。1行も触らない）
新UIは **既存のデータとエンジンを"読み書きするだけ"**。下記は挙動なので変更禁止：
- コード生成：`buildCode` / `genBlock` / `genChain` / `lib/codegen.ts`
- ブロック構造/配置/接続：`lib/blockGraph.ts` / `findSnap` / `attach` / `detach` / `getPos` / `getDepth` / `addBlock`
- データモデル：`CBlock` の形（`app/editor/_types.ts`）と `data/templates.ts` の構造
- 「マイクラへ→コード誕生」演出（`CodeRevealOverlay` / `reveal` 系）は**活かす**（壊さない）
- ＝新しいのは「見せ方・並べ方・選び方（スロット/ソリティア/右の操作）」だけ。中身の意味とコード化は既存のまま。

## 6. ファイル分担（同ファイル同時編集を避ける）
- **ヒマワリ**：`LogicPanel.tsx`（SPROUTの新UI）／`ModelPanel.tsx`・`SettingsPanel.tsx`（簡単=SPROUT/プロ=GROVE 化）／`grapeIcons.tsx` 等の素材。
- **シオン**：`GrapePanel.tsx`(GROVE)／`app/editor/page.tsx`（タブの新ナビ＝脱・切替ボタン＋**起動タブ削除はここで対応**）／Java。
- ⛔ ヒマワリは `page.tsx` / `GrapePanel.tsx` を触らない。シオンは `LogicPanel`/`ModelPanel`/`SettingsPanel` を触らない。

## 7. 進め方
1. まず**プロトタイプ**（左スロット1個＋中央に1枚置く＋右ボタン1個）で手触り確認。
2. ロジック無傷を都度 **ビルド確認**（ユーザー環境で `npm run build`。ヒマワリ環境はnpm無し）。
3. 「これ見た目？挙動？」迷ったら**手を止めてシオン経由で伊波さんに確認**。
4. SPROUTの2テーマ版(古い)は tag `tsumiki-v1`＋bundle に保全済み。安全に作り替えてよい。
